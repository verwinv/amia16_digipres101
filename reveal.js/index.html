Things Non-Coders Can Do (1)

    Flesh out user stories to exhaustively detail a specific function or workflow of the product.

    Create and/or tweak the interface design (Photoshop/Illustrator or simple paper prototyping)

    Test the product in all possible browser/OS combinations and write bug reports.

    Clean available data and export it to exchangeable formats (e.g. CSV).

Feedback

    Try to make the problem specific but concise

    Explain exactly what you were doing to cause the error to occur

    Make sure to mention if you are able to make the bug repeat, or if it only happened once and not again

    Assign to person (if you know who should fix)

    Be brave, kind, assertive
